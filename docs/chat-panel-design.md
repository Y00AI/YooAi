## 1. 布局方案（重要修正）

### ❌ 错误理解
不要修改原有面板的内部结构，不要添加 TAB 切换。

### ✅ 正确布局
```
┌──────────────────┬──────────────────────────┐
│  左侧：原有面板    │  右侧：新增聊天窗口        │
│  （完全不动）      │  （全新区域）             │
│                  │                          │
│  - 连接状态       │  - 消息列表（可滚动）     │
│  - Agent Mood    │  - 用户消息：右对齐       │
│  - Agent Soul    │  - AI 回复：左对齐        │
│  - Token 计数    │  - 支持 Markdown         │
│  - 进度条        │                          │
│  - 活动日志      │  ┌────────────────────┐  │
│  - Brain Memory  │  │ 输入框             │  │
│                  │  │ (多行)             │  │
│                  │  └────────────────────┘  │
│                  │  [发送] Cmd+Enter        │
└──────────────────┴──────────────────────────┘
        ↑                    ↑
   完全不动              全新区域
```

### 布局实现
- 使用 CSS Flexbox 或 Grid
- 左侧面板：`flex: 0 0 350px`（固定宽度，保持原样）
- 右侧聊天：`flex: 1`（自适应剩余空间）
- 中间分隔线：可拖拽调整宽度（CSS resize）

## 2. 代码拆分（重要）

### 当前问题
所有代码都在 `public/index.html` 一个文件中（1000+ 行），难以维护。

### 目标结构
```
public/
├── index.html          # 主页面（精简，只保留结构）
├── css/
│   ├── main.css        # 主样式（原有样式）
│   └── chat.css        # 聊天窗口样式
└── js/
    ├── app.js          # 主逻辑（原有功能）
    ├── chat.js         # 聊天功能
    └── gateway.js      # WebSocket 连接管理
```

### 拆分原则
1. **CSS 拆分**
   - `main.css`: 原有的所有样式
   - `chat.css`: 聊天窗口相关样式

2. **JS 拆分**
   - `gateway.js`: WebSocket 连接、消息收发
   - `chat.js`: 聊天 UI 渲染、消息处理
   - `app.js`: 主入口，整合所有模块

3. **HTML**
   - 只保留 DOM 结构
   - 引入外部 CSS 和 JS
